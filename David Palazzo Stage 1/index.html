Stage 1 Unit 1: Basics of the Web, HTML and the Internet

How It All Works

Invented in the 90s, the Web has grown to host over 30 billion pages. The vast majority of the Web is made up of inter-linked hypertext documents written in a language called Hypertext Markup Language, or HTML. When you visit a website (www.example.com) you're actually sending an HTTP request from your computer to a server who in turn finds the appropriate HTML document and sends it back to your computer where your web browser interprets and displays it. The internet is a global network of computers that connects billions of users worldwide. This video does a good job of explaining more.


HTML

HTML Hypertext Markup Language makes up most of the content on the web. HTML dictates Text Content, what you see, Markup,what it looks like behind the scenes References to other documents, images and videos Links to other pages in the form of hyperlinks. Plain text equals plain text, markups do everything else. This video gives a good overview.


Tags and Elements

HTML is written in sentences or elements composed of tags and text. Tags are the markers that determines how the content appears.
This video explains it well. A tag can be either inline or block depending on its function.

Stage 1 Unit 3: Adding Style with CSS

What is CSS?

CSS, or Cascading Style Sheets, is code written to control the style of HTML elements. Think of it as the decorations or the style of your house.

Syntax and Rules

In CSS you will first write a selector that defines what elements of the page a particular style will apply to. Choose one HTML tag or class and alter the behavior of all matching tags on the page. If, for example, you wanted to alter all of the text in div class="units" you would type "div.units {and fill in your commands here}" All phrases are written as follows {attribute:value;} Everything tagged as a "unit" will now act the same way. To refer to a "class" just add a period and the class name (ex. class=footer becomes .footer{:;})

Avoiding Repetition

Avoiding Repetition is essential in writing good code for many reasons. Here are three:

Avoiding Errors-An automated system is far less likely to fail. Most mistakes are due to human error, remove it.

Consistency-By working from generic rules down to specifics, the chances that your website will maintain the same style and sizes throughout are much higher.

Efficiency-Any time you can cut out repetition, menial tasks, time wasters, etc. you will save time, and we all know that time is money.

Stage 2 Unit 2: Variables in Python

What is a Variable?

Variables give programmers a way to give names to values. In Python, you can create a name to take the place of a number or variable by using an assignment statement. (name=expression) so (speed_of_light = 299792458) means when we print speed_of_light, 299792458 will come out. Variable can be invaluable when building complex systems or tracking data. They improve code readability by using names that make sense to humans. They give us a way to store the value of important data. They give us a way to change the value of something (like in the line days = days-1 If my_variable is a variable with a value of 2, then the following code would print out 4: print my_variable + my_variable

Assigning a Variable

In the line 2 + 3 = 5, the equals sign means "is the same as". In the line my_variable = 5, the equals sign means "takes the value of". It may start counting up or down, or could do something different all together based on the process. You can assign any name to a variable containing letters, numbers, or underscores so long as the first character is NOT a number. We can assign the value 2 to the variable my_variable by writing code like this: my_variable = 2 We can even change the value of a variable by re-assigning it to a different value later.

Stage 2 Unit 3: Functions in Python

What is a Function?

A function is something that takes input, does something to that input, and then produces output. For example, a function named "square" might take a number as input and produce the square of that number as output.

Making vs Using

Functions are made by starting a line of code with the keyword def and then giving a function name followed by the function parameters in parentheses. These parameters will eventually be replaced by actual values when the function is used (called).In the "body" of the function, we write the code that specifies what to do with the input parameters. For example the following code could be the definition of a function called "square":

def square(x):
answer = x * x
return answer

To call a function, we write the name of the function followed by the value(s) we want to give it in parentheses. Like this:
print square(4)
which will replace the "x" in the definition with the value we've input.

Importance of Functions

Functions are crucial in helping programmers avoid repetition. Not only can you automate a process, but once you define a program it saves it so you don't have to redefine programs over and over. Complex systems can be built on a series of simple functions.

Anatomy of a 'Function Definition'

On the first line of your function definition you must begin the line with
"def (lower case)"
After def you must give a function name
"def function_name (a,b):"
Next, you must have a set of parentheses with the required parameters inside. The line must end with a : colon.

In the body of the function every line in the function must be indented.
If you want your function to produce output, it must end with a return statement.

To "call" the function...
"Calling" a function just means using it. You do this by writing:
printfunction_name (x,y) <-- with input>

Decision and Repetition: If and While Statements

In most computer programming languages, a while loop is a control flow statement that allows code to be executed repeatedly based on a given boolean (T/F) condition. The while loop can be thought of as a repeating if statement.

The while construct consists of a block of code and a condition/expression.The condition/expression is evaluated, and if the condition/expression is true, the code within the block is executed. This repeats until the condition/expression becomes false. Because the while-loop checks the condition/expression before the block is executed, the control structure is often also known as a pre-test loop.

while(example_name):
(code or other complicated things)
if (break_test):
break, move on

Strings vs Lists

String - sequence of characters surrounded by single or double quotes. Can be stored in a variable by using an assignment.
" = expression"

List - sequence of anything (characters, strings, numbers, even other lists) surrounded by square brackets and elements are separated by commas. Can be stored in variables by using assignment.
= [, , ]"
Lists support mutation.

Nested Lists

Nested Lists - lists can contain many different elements, including other lists.


Mutation & Aliasing

Mutation means that we can change the value of a list after we’ve created it, modifying the existing object. With a string we can only create new strings with different values, not modify the original string.With lists we can use mutation to change the value of the list to replace the list without creating new elements.
Aliasing means we have multiple ways to refer to the same object.

Plus vs Append
